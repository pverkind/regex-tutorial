3. The dot metacharacter and escaping

In regular expressions, the dot . is a wildcard character. It stands for "any single character". Remember that for a computer, any element in a text is a character: so, the dot will match any single letter, digit, space, punctuation mark, or other symbol. There is only one exception: the dot will not match a newline character (that is, the character inserted when you press the Enter key on your keyboard).

Wait a minute - if the dot matches any character, is there no way to match a dot in a text, specifically? For example, it is easy to imagine that you'd want to match a full stop at the end of a sentence, or the decimal separator in a number like 12.345... Of course this is possible! Any of the 14 metacharacters in regular expressions can be reduced to its literal meaning by "escaping" it, that is, by putting a backslash character (\) in front of it. So, the regex \. will only match a dot in the string, not any other character; the regex \? will only match a question mark in the string; and \\ will match only a backslash.

In the search box below the sample strings, write a pattern that matches only those sequences of characters that end with a full stop.

A ✔ symbol will be displayed next to a task if your regex pattern fulfills that task.

If the task is to "match this string entirely", the ✔ symbol will appear as soon as your pattern matches the entire string. If the task is "Don't match this string", the ✔ symbol will be displayed as long as your pattern doesn't match the (entire) string.

A ✘ sign will show you in which strings your pattern did not (yet) fulfill the task.

Once your regex pattern correctly fulfills the task in every string, you can move to the next lesson.
